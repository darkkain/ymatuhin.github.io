После анализа требований к front end разработчикам в топовых компаниях, расставим приоритеты, накидаем план действий, добавим полезные ссылки и советы.

Я думал что список сильно изменится, но он почти такой-же как и в предыдущей статье, если отсортировать по упоминаниям. Но есть изменения:

  1. Английский, для меня, важен для профессионально и личного роста (не только для чтения тех. документации). Поэтому он однозначно вырывается в топ 1.
  2. Тонкости JavaScript я знаю на достаточном уровне, да и пригождаются они в только на собеседованиях (а первая задача подвинуть блок на 1px 😀). С общим пониманием у меня хорошо, недавно осилил книгу с носорогом.
  3. Gulp так-же поднимается в рейтинге, поскольку без него сложно собирать нормальный front end и пользоваться всякими ES6 ES2015.
  4. CommonJS для меня удобнее чем AMD (с которым я знаком и так), поэтому я выбираю его.
  5. React.js, выигрывает в этой битве второго фреймворка (первый AngularJS), поскольку очень популярен сейчас. Плюс Webpack частично применяется вместе с ним.
  6. Git выходит из списка, поскольку базовые знания имеются. Остается только разобраться как переносить комиты с веток т.д. (но это не критично)
  7. jQuery так-же выходит из списка по тем-же причинам.

# Итоговый список вместе с ресурсами для изучения

  1. Английский язык — спросив совет у разработчиков в твитере, я выбрал — Efаortless English (A.J. Hoge) в этот момент, мой друг запускает недельную серию вебинаров по английскому 🇬🇧, я записался)
  2. REST API, HTTP — restapitutorial.com, перевести ответ с so и поискать статьи на хабре
  3. OOP, MVC, MVVM в JavaScript — пока нет идей, буду искать в гугле
  4. Gulp — начну с курса loft blog, дальше читать и тренироваться
  5. CommonJS — прочитать доку и применить в проекте
  6. Тестирование JavaScript (unit, TDD, BDD) — ссылок в сохраненных у меня нет, буду гуглить 😊
  7. AngularJS — курс loft blog, прочитать документацию и туториолы, написать модуль
  8. JSDOC — так-же пока нет идей, гуглить и пытаться применить
  9. React.js — курс loft blog
  10. Node.js — книга «Node.js для начинающих», курс «Специалиста»
  11. Паттерны / алгоритмы и структуры данных — проанализировать JavaScript Patterns Collection, еще раз прочитать «Паттерны для масштабируемых JavaScript-приложений»

Если у вас есть ссылки, пожалуйста, поделитесь ими в комметариях. Вы поможете мне и читающим эту статью людям 👍

По мере изучения я буду писать статью для закрепления материала и чтобы меня поправили, если я неправильно понял.
